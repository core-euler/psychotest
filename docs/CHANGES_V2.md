## 1. Замена платёжной системы: Prodamus → YooKassa

### Что было
- Платёжная ссылка через Prodamus (`PRODAMUS_PAYMENT_URL`)
- Webhook endpoint: `POST /prodamus/webhook`
- Подпись: HMAC-SHA256 через `PRODAMUS_WEBHOOK_SECRET`
- Конфигурация: `PRODAMUS_PAYMENT_URL`, `PRODAMUS_WEBHOOK_SECRET`, `PRODAMUS_KEY`, `PAYMENT_STUB_MODE`

### Что стало
- Платёжная система: **YooKassa** (ЮKassa)
- Реквизиты:
  - `shopId`: `1300251`
  - `secret_key`: `live_Aj9g3IEuRbtM3QyIxVKafNmcXUlTYKAa5efoQOuexpc`
- Интеграция: создание платежа через YooKassa API → получение `confirmation_url` → пользователь переходит и оплачивает → YooKassa шлёт webhook-уведомление
- Webhook endpoint: `POST /yookassa/webhook`

### Затронутые файлы
- `bot/config.py` — заменить переменные Prodamus на YooKassa (`YOOKASSA_SHOP_ID`, `YOOKASSA_SECRET_KEY`)
- `bot/webhooks/prodamus.py` → `bot/webhooks/yookassa.py` — новый обработчик webhook
- `bot/services/messaging.py` — генерация платёжной ссылки через API вместо статического URL
- `bot/keyboards/payment.py` — кнопка «Оплатить» ведёт на динамически созданную ссылку
- `.env` / `.env.example` — обновить переменные окружения
- `requirements.txt` — добавить `yookassa` SDK

### Переменные окружения (новые)
| Переменная | Обязательна | Описание |
|---|---|---|
| `YOOKASSA_SHOP_ID` | Да | ID магазина YooKassa |
| `YOOKASSA_SECRET_KEY` | Да | Секретный ключ YooKassa |

### Удаляемые переменные
- `PRODAMUS_PAYMENT_URL`
- `PRODAMUS_WEBHOOK_SECRET`
- `PRODAMUS_KEY`
- `PAYMENT_STUB_MODE`
